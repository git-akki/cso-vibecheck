# Anchor incidents — cite these for credibility

## Moltbook (Feb 2026)
Vibe-coded social network. Hardcoded Supabase anon key in client JS bundle + Row Level Security disabled on every table. Three days post-launch, attacker dumped 1.5M auth tokens + 35K emails. Root cause: anon key in client is fine ONLY when RLS is enabled. Vibe-coding tools default RLS off "because it broke the queries".
**Cite for:** Check 6 (key in client), Check 13 (RLS).

## Lovable CVE-2025-48757 (Mar-Apr 2026)
Apps generated by Lovable shipped without RLS by default. BOLA on `/api/projects/[id]` — any authenticated user could read any project. Vulnerability disclosed via HackerOne, closed by Lovable without escalation, left open 48 days. Affected ~18,000 user records. Lovable initially called it "intentional behaviour".
**Cite for:** Check 12 (BOLA / IDOR), Check 13 (RLS), Check 19 (middleware coverage).

## Bolt.new sk- in JS bundle (ongoing)
Bolt.new frequently emits OpenAI/AWS/Google keys directly in generated client code. Patterns visible in production bundles: `sk-`, `AKIA`, `AIzaSy`. Wiz Research: ~20% of vibe apps risky, 11% of Supabase apps leak service keys.
**Cite for:** Check 6, Check 16.

## Cursor .env in git (Sherlock Forensics, 2026)
Cursor workflows encourage pasting API keys for "quick testing"; keys land in `.cursor/` config or get committed to git. 78% of AI-built apps have exposed env files in assessments.
**Cite for:** Check 16.

## Snake game pickle RCE (Claude, multiplayer build)
Claude wrote a multiplayer snake game; network layer used Python pickle for object serialization. pickle.loads on network input = arbitrary RCE.
**Cite for:** Check 18.
